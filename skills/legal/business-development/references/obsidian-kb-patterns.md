# Obsidian 多文档知识库创建模式

当用户要求在 Obsidian 中创建某个领域的知识库时，使用以下结构模式。

## 标准目录结构

```
XX-领域名称/
├── README.md              ← 导航中心（MOC）
├── 核心文档1.md
├── 核心文档2.md
└── 子目录/
    ├── 子文档1.md
    └── 子文档2.md
```

## README.md（导航中心）模板

```markdown
# 📺 [领域名称]知识库

> 一句话说明知识库用途

---

## 🎯 核心目标

- 目标1
- 目标2

---

## 📂 知识库结构

### 子主题A
- [[文档1]] - 简要说明
- [[文档2]] - 简要说明

### 子主题B
- [[文档3]] - 简要说明

---

## 🔗 相关链接

- [[其他知识库导航]]
```

## 文档间链接规范

- 使用 `[[文档名]]` 语法，不带路径和 `.md` 后缀
- 每个文档底部放置"相关文档"区块，列出关联文档
- 导航中心使用分类分组，每组3-5个文档

## Excalidraw 可视化知识图谱模式

当知识体系涉及复杂关系（如五行生克、十神体系）时，使用 Excalidraw 创建可视化图表辅助记忆。

### 使用方式

1. 创建 `.excalidraw` 文件保存在知识库目录
2. 使用 `excalidraw` skill 的 `scripts/upload.py` 上传获取分享链接
3. 在 Obsidian 中可直接嵌入 `.excalidraw` 文件

### 图表类型

| 图表类型 | 适用场景 | 布局 |
|----------|----------|------|
| 流程图 | 学习路径、工作流程 | 左→右或上→下 |
| 关系图 | 概念关系、生克关系 | 中心辐射 |
| 场景图 | 业务应用、对话示例 | 分区卡片 |
| 速记图 | 知识点速查 | 网格/表格 |

### Excalidraw 创建要点

- 使用 `containerId` + `boundElements` 绑定文字到形状（不要用 `label` 属性）
- 每个元素的 `boundText` 紧跟其容器元素后面
- 颜色使用 skill 的 `references/colors.md` 中的调色板
- 最小 `fontSize: 16`，标题用 `fontSize: 20-32`
- 不要在文字中使用 emoji（Excalidraw 字体不支持）

## 关键原则

1. **先创建目录结构，再填充内容** — 用 `mkdir -p` 创建目录
2. **README 是入口** — 用户从这里开始浏览
3. **文档自包含** — 每个文档独立可读，但通过链接串联
4. **表格优先** — 能用表格呈现的用表格
5. **使用 Obsidian vault 路径** — `/Users/yuzhenghong/Library/Mobile Documents/iCloud~md~obsidian/Documents/法律知识库/`
6. **Excalidraw 可视化** — 复杂知识体系配可视化图表，上传后在文档中附链接
