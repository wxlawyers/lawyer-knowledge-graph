# 律师微信公众号文章撰写技能

## 触发条件

用户要求撰写/修改/优化微信公众号法律科普文章时加载本技能。

## 一、选题护城河策略

### 用户身份标签
- **执业年限**：15年资深律师
- **执业机构**：江苏三诚律师事务所（合伙制）
- **执业领域**：民商事诉讼、刑事诉讼、知识产权、公司法/商事合规
- **管辖地域**：江苏省内（海门、宜兴、无锡、南通等）

### 选题公式（护城河）
```
15年执业经验 × 江苏本地法院裁判 × 企业家视角 = 独一无二
```

### 蓝海选题池
| 选题方向 | 护城河理由 | 竞争度 |
|---------|-----------|--------|
| 新公司法实缴 | 企业家刚需+政策热点 | 低 |
| 股东纠纷 | 商事诉讼核心，普通人不讲 | 低 |
| 合同陷阱 | 15年踩坑经验，AI写不出 | 低 |
| 江苏法院裁判规则 | 地域壁垒，外地抄不了 | 极低 |
| 刑民交叉 | 刑辩+商事双背景，稀缺 | 极低 |
| 知识产权实操 | 知产+商事结合，不是理论派 | 低 |

### 红海选题（避开）
- ❌ 婚姻家事（出轨、离婚、抚养权）——情绪化、低俗化
- ❌ 劳动纠纷（N+1、加班费）——烂大街
- ❌ 债务/征信（信用卡协商、反催收）——灰产关联
- ❌ 交通事故——同质化严重
- ❌ 法条搬运——AI批量生产

### 每篇选题必须包含
1. ✅ **真实案例**（脱敏）——不是编的
2. ✅ **江苏法院裁判**——地域壁垒
3. ✅ **企业家痛点**——高净值客户共鸣
4. ✅ **实操建议**——不是法条翻译

## 二、19条质量标准（必须严格执行）

### 数据与权威性
1. **交叉对比权威数据**：北大法宝+元典智库+权威媒体，多源验证
2. **律师权威性**：专业术语准确、分析有深度、不浮于表面
3. **数据标明出处**：引用数据必须注明来源（如"据最高法2025年工作报告"）
4. **能溯源**：法条给条文号，案例给案号，数据给出处链接

### 降AI味
5. **降AI生成味**：口语化表达，有个人观点，有"我执业15年"的亲身经历感
6. **禁止AI替代真人创作**（2025新规）：轻度限流，重度封号

### 文字与校对
7. **文字不能写错**：逐字校对，特别是法条、案号、金额
8. **插图内容真实**：不使用AI生成虚假图片
9. **插图文字校对**：图中文字与正文一致，无错别字

### 引用规范
10. **法条引用格式**：全称+条号+内容，如"《中华人民共和国公司法》第四十七条"，不能简写"公司法47条"
11. **案例引用格式**：案号+法院+年份，如"（2025）苏01民终XXX号"，脱敏处理当事人信息
12. **转载不侵权**：原创为主，引用标注出处，不搬运他人文章

### 法律内容特殊要求
13. **免责声明**：文末加"本文仅供学习交流，不构成法律意见，具体案件请咨询律师"
14. **敏感信息脱敏**：当事人以"甲""乙"代称，不出现真实姓名、身份证号、地址
15. **不得承诺案件结果**
16. **不得夸大宣传法律服务效果**

### 排版与发布
17. **排版规范**：短段落（3-4行）、多用小标题、关键信息加粗、适当用表格
18. **发布时间**：工作日早上7:30-8:30（通勤时间）或晚上20:00-21:00（休息时间）
19. **封面图**：文字型封面，不用AI人像
20. **互动引导**：文末引导"点赞+在看+关注"，不做强制诱导（违规红线）
21. **关键词布局**：标题含核心关键词，利于微信搜一搜
22. **声明原创**：每篇声明原创，防止被搬运

## 三、文章结构模板

### 标准结构（400-800字）
1. **开头**：用一个真实案例引入（3句话让读者代入）
   - 示例："去年我接了一个案子，海门一个老板甲总来找我……"
2. **痛点**：问题是什么，风险有多大
3. **法律依据**：引用具体法条（全称+条号）
4. **真实案例**：江苏法院裁判，标注案号
5. **实操建议**：3条以内，企业家听得懂
6. **金句总结**：一句话记住
7. **免责声明**+**作者信息**+**关键词标签**+**互动引导**

### 标题公式
- ✅ 痛点+数字：《注册资本填太高，老板差点坐牢》
- ✅ 反常识+悬念：《签合同最容易漏掉的一个条款》
- ❌ 避免：震惊！必看！99%的人不知道

## 五、HTML排版导出

### 公众号文章HTML规范
- 最大宽度680px，白色背景，圆角阴影
- 标题居中，摘要蓝色边框
- 风险提示：橙色警示框
- 案例：绿色案例框
- 法条引用：蓝色斜体框
- 核心金句：渐变紫色背景
- 三条建议分点突出

**⚠️ 铁律：公众号编辑器HTML兼容性规则（2026-06-07 / 2026-06-10 教训）**

公众号编辑器对HTML/CSS支持极其有限。2026-06-10实测发现：即使完全使用内联样式、不用class、不用table、不用外部CSS，以下CSS属性仍然**不生效或导致内容不显示**：

| 不支持的CSS属性 | 说明 |
|------|------|
| `border-radius` | 圆角，不生效 |
| `box-shadow` | 阴影，不生效 |
| `rgba()` | 半透明颜色，不生效 |
| `linear-gradient()` | 渐变背景，导致整个div不显示 |
| `transform` | 变换，不生效 |
| `transition` | 动画，不生效 |
| `flex` / `grid` | 布局，不生效 |

**安全的CSS属性（仅这些可用）**：
- `background`（纯色，如 `#1a365d`）
- `color`
- `padding` / `margin`
- `border` / `border-left` / `border-top` 等
- `font-size` / `font-weight`
- `text-align`
- `line-height`

**终极方案：纯文本格式**
如果HTML粘贴后仍不显示，直接转为纯文本：
- 用 `【标题】` 代替 `<h2>` 标题
- 用 `【加粗】` 代替 `<strong>` 加粗
- 用空行分隔段落
- 粘贴到公众号编辑器后，用编辑器自带的格式工具重新排版

**纯文本转换脚本**：`scripts/html-to-plaintext.py`
```bash
python3 scripts/html-to-plaintext.py input.html output.txt
```

**错误做法（公众号中不显示）**：
```html
<div class="summary">
    <table>
        <tr><th>条款</th><th>要点</th></tr>
        <tr><td>法律适用</td><td>适用哪国法律</td></tr>

```

    <p style="color: #fff; margin-bottom: 15px;">核心要点：适用哪国法律</p>

```

```

### 插入图片

**⚠️ 铁律：MEDIA: 协议只在 Hermes 内部有效，浏览器打不开（2026-06-10 教训）**

HTML文件在浏览器中预览时，`MEDIA:` 前缀的图片无法加载。必须用 base64 内嵌：

```python
import base64
with open("image.png", "rb") as f:
    b64 = base64.b64encode(f.read()).decode()
# 在HTML中：
# <img src="data:image/png;base64,{b64}" ...>
```

注意：base64 会使 HTML 文件膨胀（1MB截图 ≈ 1.3MB base64）。如用户要求去掉插图，需同时删除 base64 内容和本地缓存文件。

**从网页截图获取插图的流程**：
1. `browser_navigate` → 打开目标页面
2. `browser_vision` → 截图并保存
3. 截图路径在 `~/.hermes/cache/screenshots/` 下
4. 用 Python 转 base64 嵌入 HTML

**清理插图时**：
1. 从 HTML 中删除 `<img>` 标签及其父 `<div>`
2. 删除 `~/.hermes/cache/screenshots/` 下对应截图
3. 删除 `~/Documents/` 下下载的原始图片

### 导出步骤
1. 生成HTML文件到 `~/Documents/`
2. `open` 命令在浏览器打开
3. 用户Cmd+A全选 → Cmd+C复制 → 粘贴到公众号编辑器
4. 如果格式丢失，分段截图上传到公众号

## 九、常见陷阱

| 陷阱 | 正确做法 |
|------|---------|
| 法条简写"公司法47条" | 必须写全称"《中华人民共和国公司法》第四十七条" |
| 案例无案号 | 必须标注案号+法院+年份 |
| 当事人真名 | 必须脱敏：甲总、何某、A公司 |
| AI味重的开头 | 用"去年我接了一个案子"开头 |
| 承诺案件结果 | 只说"有补救余地"，不说"一定能赢" |
| 诱导分享 | 只说"觉得有用点个赞"，不做"转发领资料" |
| 封面用AI人像 | 用文字型封面 |
| 数据无出处 | 标注"据北大法宝案例数据库"等 |
| 用户提供的数字未核实 | 发布前必须让用户确认所有关键数字（如"11亿"vs"110亿"） |
| 图片用MEDIA:协议 | 浏览器打不开，必须转base64内嵌 |
