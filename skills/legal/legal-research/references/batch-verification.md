# 批量法条核实工作流

## 适用场景

网站、知识库、文档中有大量法条引用需要核实正确性时使用。

## 工作流程

### 步骤一：提取法条引用

从文件中自动提取所有法条引用：
```python
import re
# 提取 "民法典第XXX条"、"公司法第XX条" 等格式
pattern = r'(?:民法典|公司法|刑法|劳动法|劳动合同法|商标法|专利法|著作权法|行政诉讼法|民事诉讼法|国家赔偿法|企业破产法|反不正当竞争法|旅游法|农村土地承包法|保险法|海商法|个人信息保护法|行政处罚法|行政许可法)第(\d+)条'
```

### 步骤二：批量验证

使用 delegate_task 并行验证（每次最多3个子任务）：

```
delegate_task(tasks=[
    {"goal": "核实前N条法条引用...", "toolsets": ["terminal", "file"]},
    {"goal": "核实中间N条法条引用...", "toolsets": ["terminal", "file"]},
    {"goal": "核实最后N条法条引用...", "toolsets": ["terminal", "file"]}
])
```

每条法条使用 `mcp_yuandian_search_fatiao(keyword="法规名 第XXX条")` 验证。

### 步骤三：输出核实报告

| 序号 | 法条引用 | 期望内容 | 核实结果 | 说明 |
|------|---------|---------|---------|------|
| 1 | 民法典第667条 | 借款合同 | ✅正确 | 实际内容匹配 |
| 2 | 民法典第919条 | 旅游合同 | ❌错误 | 实际是委托合同定义 |
| 3 | 旅游法第65条 | 解除权 | ⚠️未找到 | MCP未返回，需人工复核 |

### 步骤四：修正错误

**发现错误后必须立即修正**：
1. 找到文件中错误的法条引用
2. 替换为正确的条文号或删除错误引用
3. 推送到远程仓库
4. 在核实报告中标注"已修正"

## 常见错误类型

| 错误类型 | 示例 | 修正方法 |
|---------|------|---------|
| 条文号张冠李戴 | 农村土地承包法第26条（承包期限）→ 实际在第21条 | 改为正确条号 |
| 引用已废止法条 | 合同法第XXX条 → 已被民法典取代 | 改为民法典对应条文 |
| 条文内容不符 | 民法典第592条写成"或者扩大" → 原文无此表述 | 用MCP获取原文逐字比对 |
| 法规名错误 | 引用"担保法" → 已被民法典取代 | 改为民法典担保制度编 |

## 间接验证法

当MCP直接搜索返回空结果时：

1. 搜索引用该条文的其他文件（司法解释、法院指引）
2. 从引用中确认条文内容
3. 如果多个独立来源引用同一内容，可确认正确
4. 如果发现引用内容矛盾，标记为需人工复核

## 输出格式

核实完成后，输出汇总：
- ✅ 正确：N条（XX%）
- ❌ 错误：N条（已修正/待修正）
- ⚠️ 未找到/需人工复核：N条
