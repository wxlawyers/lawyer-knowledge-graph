# 法律检索与案例研究技能

面向中国诉讼律师的法律检索框架，集成元典智库和北大法宝MCP工具，提供三轮递进检索策略。

## 适用场景

- 请求权基础检索
- 类案检索
- 裁判规则验证
- 法规时效性确认
- 法条内容查询

## MCP 工具

### 元典智库（5个工具）

| 工具名称 | 功能 | 用途 |
|----------|------|------|

| mcp_yuandian_search_fatiao | 法条检索 | 精确检索法条内容 |
| mcp_yuandian_search_qwal | 案例检索 | 检索司法案例 |

## 执行步骤

### 步骤一：确定检索目标

#### 检索类型
1. **请求权基础检索**
   - 确定法律关系性质
   - 检索相关法规
   - 检索具体法条

2. **类案检索**
   - 确定案由和争议焦点
   - 检索类似案例
   - 提取裁判规则

3. **裁判规则验证**
   - 验证规则适用性
   - 对比我方案情
   - 判断参考价值

#### 检索关键词
1. **案由关键词**
   - 合同纠纷
   - 侵权纠纷
   - 物权纠纷
   - 公司纠纷

2. **争议焦点关键词**
   - 违约责任
   - 侵权责任
   - 物权变动
   - 股东权利

3. **地域关键词**
   - 省份名称
   - 城市名称
   - 法院名称

4. **时间关键词**
   - 年份
   - 时间段

### 步骤二：第一轮检索（双源交叉）

#### 元典智库检索

**法规检索**
```
mcp_yuandian_search_fagui(keyword="民法典")
```

**法条检索**
```
mcp_yuandian_search_fatiao(keyword="民法典 第五百七十七条")
```

**案例检索**
```
mcp_yuandian_search_qwal(keyword="合同纠纷 违约 南京 2024")
```

#### 北大法宝检索

**法规搜索**
```
mcp_pkulaw_law_get_law_list(title="民法典")
```

**案例检索**
```
mcp_pkulaw_case_search_search_case(text="合同纠纷 违约")
```

**法条检索**
```
mcp_pkulaw_get_article(title="民法典", number="五百七十七条")
```

#### 交叉验证
1. **对比两个平台的检索结果**
   - 法规是否一致
   - 法条内容是否一致
   - 案例是否一致

2. **标注差异**
   - 记录不同之处
   - 分析原因
   - 确认权威来源

### 步骤三：第二轮检索（裁判规则验证）

#### 提取裁判规则
1. **从案例中提取**
   - 裁判要旨
   - 裁判理由
   - 裁判结果

2. **归纳规则**
   - 提炼核心规则
   - 确定适用条件
   - 明确法律依据

#### 验证规则适用性
1. **对比案情**
   - 事实是否相似
   - 法律关系是否相同
   - 争议焦点是否一致

2. **判断参考价值**
   - 案例级别（最高法/高级法院/中级法院）
   - 案例时间（是否最新）
   - 案例地域（是否同一地区）

### 步骤四：第三轮检索（类案补充）

#### 检索策略
1. **同一法院历史裁判**
   - 检索同一法院的类似案例
   - 分析裁判倾向
   - 预判裁判结果

2. **同一法官历史裁判**
   - 检索同一法官的类似案例
   - 分析裁判风格
   - 预判裁判倾向

3. **同一地区中级法院**
   - 检索同一地区中级法院案例
   - 分析裁判标准
   - 预判上诉结果

#### 检索关键词组合
1. **案由 + 争议焦点**
   - 合同纠纷 + 违约责任
   - 侵权纠纷 + 因果关系
   - 物权纠纷 + 物权变动
   - 公司纠纷 + 股东权利

2. **案由 + 地域 + 年份**
   - 合同纠纷 + 南京 + 2024
   - 侵权纠纷 + 江苏 + 2023
   - 物权纠纷 + 无锡 + 2024

3. **争议焦点 + 法律依据**
   - 违约责任 + 民法典第五百七十七条
   - 侵权责任 + 民法典第一千一百六十五条
   - 物权保护 + 民法典第二百三十五条

### 步骤五：检索结果处理

#### 法规检索结果
1. **确认时效性**
   - 是否现行有效
   - 是否被修改
   - 是否被废止

2. **确认完整性**
   - 条文是否完整
   - 是否有司法解释补充
   - 是否有指导性案例

3. **标注来源**
   - 检索平台
   - 检索时间
   - 检索关键词

#### 案例检索结果
1. **确认有效性**
   - 是否被撤销
   - 是否被改判
   - 是否被再审

2. **提取裁判要旨**
   - 核心裁判观点
   - 裁判理由
   - 法律依据

3. **判断参考价值**
   - 案例级别
   - 案例时间
   - 案例地域

#### 法条检索结果
1. **确认内容完整性**
   - 条文是否完整
   - 是否有司法解释补充
   - 是否有指导性案例

2. **标注来源**
   - 检索平台
   - 检索时间
   - 检索关键词

## 检索策略

### 策略一：请求权基础检索

**目的**：确定请求权基础的法律依据

**步骤**：
1. 确定法律关系性质（合同/侵权/物权/公司）
2. 检索相关法规（民法典合同编/侵权责任编/物权编/公司法）
3. 检索具体法条（违约责任/侵权责任/物权保护/股东权利）
4. 交叉验证法条有效性

### 策略二：类案检索

**目的**：查找支持我方主张的类案

**步骤**：
1. 确定案由和争议焦点
2. 组合关键词检索（案由+争议焦点+地域+年份）
3. 提取裁判要旨
4. 验证案例有效性

### 策略三：裁判规则验证

**目的**：验证裁判规则的适用性

**步骤**：
1. 检索相关案例
2. 提取裁判要旨
3. 对比我方案情
4. 判断是否适用

## 质量标准

### 检索质量
1. **全面性** — 是否覆盖所有相关法规和案例
2. **准确性** — 检索结果是否准确
3. **时效性** — 法规是否现行有效
4. **权威性** — 案例是否具有参考价值

### 引用规范
1. **法条引用** — 全称+条号+内容
2. **案例引用** — 案号+法院+裁判要旨
3. **来源标注** — 检索平台+检索时间

## MCP 工具可靠性与降级策略

### 已知问题（2026-06 实测）
1. **北大法宝 MCP 服务不稳定**：`pkulaw-case-search`、`pkulaw-law`、`pkulaw-case` 等服务可能返回 "MCP server not connected" 错误。这是 HTTP 服务端连接问题，非本地配置问题。
2. **元典智库案例检索可靠性极差（重要！）**：`search_qwal` 对非主流案由（如建设工程施工合同、知识产权等）的检索**完全失效**——无论关键词如何组合，返回的都是不相关的案例（如股权转让、著作权等）。实测多轮检索全部失败。该工具仅对极少数高频案由可能有效，不能依赖。**2026-06-09实测补充**：`search_qwal` 对基于日期的查询（如"2026年6月 典型案例"）同样失效——返回的是2007-2010年的旧案例，与日期完全无关。每日速报中不能依赖此工具获取"最新案例"，必须通过浏览器采集最高法/最高检官网。
3. **元典智库法规检索较可靠**：`search_fagui` 和 `search_fatiao` 对法规、司法解释的检索效果较好，能找到地方法院审判指导意见等文件。
4. **MCP工具无法覆盖地方法院指导文件**：省级高院的审判指南、审理指南、委托鉴定工作指南等文件，MCP数据库可能不收录。需通过法院官方网站（如 jsfy.gov.cn）浏览器搜索获取。

### 降级策略（MCP 不可用时）
当北大法宝 MCP 服务不可用时，按以下优先级降级：

1. **优先使用元典智库**：`search_fagui`（法规）、`search_fatiao`（法条）仍可用且可靠
2. **Web 抓取官方来源**：直接访问权威网站获取最新信息（见 `references/authoritative-sources.md`）
3. **标注降级状态**：在检索报告中注明"MCP服务不可用，数据来源为官网直采"

### 每日法律速报采集流程
当执行每日法律速报任务时，推荐以下流程：
1. **并行抓取官方来源**（browser_navigate）：最高法、最高检、司法部、江苏法院
2. **MCP 补充验证**：用元典智库 `search_fagui` 检索近期法规动态
3. **交叉比对**：将 MCP 结果与官网抓取结果比对
4. **生成报告**：按 Obsidian 格式输出 + Excalidraw 配图

## Pitfalls

### 0. 绝对禁止编造法律内容（最高优先级铁律！2026-06-09血泪教训）
**错误**：凭记忆编造案例名称、案号、裁判要旨，或编造司法解释条文内容
**正确**：
- **案例**：必须通过MCP工具检索到真实案例，附带案号、法院、裁判日期
- **法条**：必须通过MCP工具获取原文
- **找不到的内容**：标注"⚠️待核实"，**绝不编造**

**2026-06-09教训**：创建医疗纠纷典型案例时，编造了"指导案例24号"内容（实际该案例是交通事故案件非医疗纠纷），编造了张某诉某医院等虚构案例。用户质疑"核实真实性了？"后不得不承认未核实。

**铁律**：**先核实，后写入**。写入任何法律内容之前，必须先用MCP工具核实。

### 1. 法条引用未用MCP工具核实（极高频！）
**错误**：凭记忆引用法条条文，未经MCP工具核实
**正确**：法条引用必须用 `mcp_pkulaw_get_article` 获取准确条文原文
**核实流程**：
1. 用 `mcp_pkulaw_get_article(title="中华人民共和国民法典", number="第XXX条")` 获取原文
2. 对比我方文件中的引用
3. 如有差异立即修正
4. 在文件中标注"来源：北大法宝"和核实日期

### 2. 引用已废止的旧法条文（高频！）
**错误**：引用旧版买卖合同司法解释第12/14/31条等（2020修正版已删除）
**正确**：
- 买卖合同司法解释2020年修正版删除了旧版第2/3/4/15/16/18/28/30/32/35/37/41/43条
- 条文顺序重新调整，旧编号不能引用
- 合同法已被民法典取代

**核实方法**：用 `mcp_pkulaw_search_article(text="关键词")` 搜索当前有效条文

### 3. 引用条文内容与原文不符（高频！）
**错误**：如民法典第592条，写成"对方对损失的发生或者扩大也有过错的"
**正确**：原文是"对方对损失的发生有过错的"（无"或者扩大"）
**验证**：必须用MCP工具获取原文后逐字比对

### 4. 批量法条核实模式（2026-06-08新增）
**场景**：网站/知识库中有200+条法条引用需要核实
**方法**：用delegate_task批量核实：
```
delegate_task(goal="核实以下N条法律依据引用的正确性。使用mcp_yuandian_search_fatiao逐条验证，输出核实结果表格。每条标注✅正确/❌错误/⚠️未找到。", toolsets=["terminal","file"])
```
**输出格式**：
| 序号 | 法条引用 | 期望内容 | 核实结果 | 说明 |
|------|---------|---------|---------|------|
| 1 | 民法典第667条 | 借款合同 | ✅正确 | ... |
| 2 | 民法典第919条 | 旅游合同 | ❌错误 | 实际是委托合同定义 |

**核实后必须立即修正错误并推送。**

### 5. 元典智库vs北大法宝选择
**元典智库**：法规检索（search_fagui）更可靠，法条检索（search_fatiao）对主流法条有效
**北大法宝**：案例检索更全面，法条精确获取（get_article）更准确
**推荐**：先用元典智库检索，再用北大法宝交叉验证

### 6. 知识库批量验证模式
当完成领域知识库沉淀后，需对全部内容做系统性验证：

**验证清单**：
1. 核心法律施行日期 → `search_fagui` 查原文确认
2. 司法解释文号 → `search_fagui` 查原文确认
3. 核心条文内容 → `search_fatiao` 获取原文逐字比对
4. 法律适用规则 → 对照法条原文确认逻辑正确
5. 实务操作（仲裁机构、贸易术语等）→ 交叉比对多个来源

**验证报告格式**：
```
| 验证项目 | 知识库记载 | 官方原文 | 验证结果 |
|----------|-----------|----------|----------|
| 法律名称第X条 | ... | ... | ✅/❌ |
```

**关键原则**：
- 简化表述需标注"简化表述"，保留核心要件
- 每个模块的核心法律引用必须一致（交叉验证）
- 发现不一致立即修正，不要等到最后

### 5. 检索时自行缩小地域范围（极高频！）
**错误**：用户要求检索"江苏省"范围的案例，代理自行缩小到"南通""无锡"等特定城市
**正确**：除非用户明确指定具体城市，否则检索范围应覆盖全省/全国，找最权威、最全的案例和指导文件
**原因**：律师管辖范围可能跨多个城市，缩小范围会遗漏重要裁判规则

### 6. 地方法院指导文件需通过法院官网获取
**问题**：省级高院的审判指南、审理指南、典型案例等文件，MCP数据库（元典智库/北大法宝）可能不收录
**正确做法**：用 `browser_navigate` 访问省高院官网（如江苏法院网 jsfy.gov.cn），在搜索框输入关键词检索
**实例**：江苏高院《建设工程施工合同案件审理指南》（2010年，约2万字）仅在江苏法院网可找到

### 7. MCP搜索返回空结果时的间接验证法
**问题**：`mcp_yuandian_search_fatiao` 对某些法条返回空结果（如旅游法、农村土地承包法、企业破产法部分条文），不能直接验证
**正确做法**：用间接验证法——搜索引用该条文的其他文件（如司法解释、法院指引、政府文件），从引用中确认条文内容
**实例**：
- 搜索"旅游法第65条"→ 空结果 → 搜索"旅游合同纠纷 审理" → 找到法院审理指南引用了第65条 → 间接确认正确
- 搜索"农村土地承包法第26条"→ 空结果 → 搜索"承包期限 三十年" → 找到第21条规定承包期限 → 发现原引用第26条错误
**关键**：间接验证既能确认正确引用，也能发现错误引用（条文号张冠李戴）
