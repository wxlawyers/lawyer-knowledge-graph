# 律师微信公众号文章撰写技能

## 触发条件
用户说"写文章""公众号""科普""法律热点"时自动进入此技能。

## 19条质量标准

| 序号 | 标准 | 要求 |
|------|------|------|
| 1 | 交叉对比权威数据 | 多源验证，不凭印象 |
| 2 | 降AI味 | 口语化，不卖弄法条，像跟朋友聊天 |
| 3 | 律师权威性 | 体现15年资深+江苏法院+商事诉讼经验 |
| 4 | 文字校对零差错 | 无错别字、标点错误 |
| 5 | 插图真实 | 不用虚假图片，用HTML截图 |
| 6 | 数据标明出处 | 注明来源和时间 |
| 7 | 能溯源 | 信息可查证 |
| 8 | 法条全称+条号 | 如《劳动合同法》第47条，不能只写"劳动法规定" |
| 9 | 案例案号+脱敏 | 脱敏处理，用"甲公司""乙某" |
| 10 | 文末免责声明 | "具体案件请咨询律师" |
| 11 | 短段落+小标题+表格 | 排版清晰，每段不超过4行 |
| 12 | 发布时间 | 7:30-8:30或20:00-21:00 |
| 13 | 封面文字型 | 文字为主，突出标题 |
| 14 | 声明原创 | 标记原创 |
| 15-19 | 护城河定位 | 15年资深×江苏法院×商事诉讼×企业家视角 |

## 护城河定位
避开婚姻/劳动/债务红海，聚焦：
- 涉外法律服务
- 商事诉讼
- 公司并购
- 知识产权

## 文章结构模板

```
标题：一句话制造好奇或痛点，20字以内

开头：用一个真实或虚构的案例引入，3句话让读者代入

正文：
- 痛点（为什么关注这个问题）
- 法律依据（法条+条号，不能只写法条名称）
- 实务要点（表格或列表形式）
- 正确做法（具体可操作的建议）

总结：核心要点回顾

免责声明：本文内容仅供法律知识交流，不构成具体法律意见

作者信息：余正洪律师 | 江苏三诚律师事务所
```

## ⚠️ PITFALL: 法条引用必须核实

**绝不能凭记忆写法条原文。** 必须用元典智库或北大法宝核实后再写。

核实步骤：
1. 用 `mcp_yuandian_search_fatiao` 检索法条
2. 确认条号和原文一致
3. 再写入文章

**2026-06-04华都琥珀案教训**：举证责任分配要视案件案由、性质而定，不能一杆子打死。法条引用必须核实原文。

## ⚠️ PITFALL: 案例不能编造

**律师网站/公众号的案例必须真实，不能编造。**

可接受的案例来源：
1. 自己代理的真实案件（脱敏后）
2. 中国裁判文书网公开案例
3. 最高法指导案例
4. 元典智库/北大法宝检索到的案例

**不能做的：**
- 编造案号
- 编造判决结果
- 编造当事人信息

## ⚠️ PITFALL: WeChat公众号HTML兼容性

**公众号编辑器不支持复杂HTML/CSS，必须用简单格式。**

### 不支持的
- CSS类选择器（`class="xxx"`）
- 复杂表格（`<table>`标签渲染异常）
- 渐变背景（`linear-gradient`）
- 外部样式表

### 支持的
- 内联样式（`style="color: #fff"`）
- 简单标签：`<p>` `<strong>` `<br>` `<span>`
- 背景色（`background: #1a365d`）
- 文字颜色（`color: #fff`）

### 总结/表格部分处理
公众号中表格经常不显示，**建议用纯文本+加粗**：

```markdown
**总结**

**1. 法律适用条款**
核心要点：适用哪国法律
避坑方法：明确约定，中英文一致

**2. 争议解决条款**
核心要点：仲裁还是诉讼
避坑方法：二选一，约定明确机构
```

### 插图制作
1. 用HTML创建插图页面
2. 浏览器打开
3. 截图保存
4. 上传到公众号

**不要尝试直接粘贴HTML代码到公众号编辑器。**

## 余律师写作风格
- 直接、务实、不绕弯子
- 用案例说话，不用法条堆砌
- 表格优先，便于对比
- 结论先行，细节在后
- 口语化但不失专业性

## 四性验证流程（发布前必做）

所有法律内容发布前必须验证：

1. **准确性** — 法条条文、案号、法院名称、日期准确无误
2. **真实性** — 引用的法规、案例真实存在，非编造或已失效
3. **合法性** — 法律依据与结论符合现行有效法律规定
4. **关联性** — 引用内容与待解决问题有实质关联

### 验证工具
```
法条 → mcp_yuandian_search_fatiao（元典智库，优先）
     → mcp_pkulaw_get_article（北大法宝，交叉验证）

案例 → mcp_yuandian_search_qwal（元典智库）
     → mcp_pkulaw_case_search_search_case（北大法宝）
```
